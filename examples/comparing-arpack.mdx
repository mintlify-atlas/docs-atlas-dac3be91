This guide shows how to compare results from Arnoldi against SciPy's `scipy.sparse.linalg.eigs`, which uses the ARPACK library.

## Why Compare with ARPACK?

ARPACK is the industry-standard eigensolver for sparse matrices. Comparing against it helps:

- Validate that Arnoldi produces correct results
- Benchmark performance and convergence speed
- Understand differences in algorithm behavior

## Side-by-Side Comparison

Here's a complete example comparing both implementations:

```python
import numpy as np
from scipy.sparse.linalg import eigs
from arnoldi.krylov_schur import partial_schur
from arnoldi.matrices import mark
from arnoldi.utils import arg_largest_real

# Markov walk matrix
A = mark(50)

TOL = 1e-8
MAX_RESTARTS = 1_000
MAX_DIMS = 20
K = 5
WHICH = "LR"
SORT_FUNCTION = arg_largest_real

# ========== Arnoldi (this implementation) ==========
Q, T, history = partial_schur(
    A,
    K,
    max_dim=MAX_DIMS,
    stopping_criterion=TOL,
    sort_function=SORT_FUNCTION,
    max_restarts=MAX_RESTARTS,
)
# Convert Schur transform into eigenpairs
vals, S = np.linalg.eig(T)
vecs = Q @ S

print("========== true residuals, our implementation =============")
for k in range(K):
    val, vec = vals[k], vecs[:, k]
    residual = np.linalg.norm(A @ vec - val * vec)
    norm_residual = residual / np.abs(val)
    print(
        f"Residual {k}, value is {np.real(val):.7}, res is {residual:.3e}, norm res is {norm_residual:.3e}"
    )

# ========== ARPACK (SciPy) ==========
vals, vecs = eigs(A, K, ncv=MAX_DIMS, maxiter=MAX_RESTARTS, which=WHICH,
                  tol=TOL)

print("========== true residuals, ARPACK (scipy) implementation =============")
for k in range(K):
    val, vec = vals[k], vecs[:, k]
    residual = np.linalg.norm(A @ vec - val * vec)
    norm_residual = residual / np.abs(val)
    print(
        f"Residual {k}, value is {np.real(val):.7}, res is {residual:.3e}, norm res is {norm_residual:.3e}"
    )
```

## Parameter Mapping

Here's how parameters map between the two implementations:

| Arnoldi | ARPACK (SciPy) | Description |
|---------|----------------|-------------|
| `K` | `k` | Number of eigenvalues to compute |
| `max_dim` | `ncv` | Maximum Krylov space dimension |
| `max_restarts` | `maxiter` | Maximum number of iterations/restarts |
| `stopping_criterion` | `tol` | Convergence tolerance |
| `sort_function=arg_largest_real` | `which="LR"` | Target largest real part |
| `sort_function=arg_largest_magnitude` | `which="LM"` | Target largest magnitude |

## Verifying Residuals

For both implementations, the residual formula is the same:

```python
residual = np.linalg.norm(A @ vec - val * vec)
norm_residual = residual / np.abs(val)
```

- **Absolute residual**: `||A*v - λ*v||` - should be small in absolute terms
- **Normalized residual**: `||A*v - λ*v|| / |λ|` - normalized by eigenvalue magnitude

Both normalized residuals should be on the order of the specified tolerance or smaller.

## Performance Comparison

To compare performance metrics like matrix-vector products and runtime, use the comparison script:

```bash
python scripts/compare-against-arpack.py path/to/matrix.mat --nev 6 --tol 1e-8
```

### Command-Line Options

- `--nev`: Number of eigenvalues to compute (default: 6)
- `--tol`: Convergence tolerance (default: 1e-8)
- `--max-dim`: Maximum Krylov space dimension (default: max(2*nev+1, 20))
- `--max-it`: Maximum number of restarts (default: 10000)
- `--which`: Target eigenvalues - "LM" (largest magnitude) or "LR" (largest real part)
- `--p`: P value in Krylov-Schur algorithm (default: k+7)

## Notes on Implementation Differences

- **Algorithm**: Arnoldi uses the Krylov-Schur method, while ARPACK uses implicitly restarted Arnoldi
- **Convergence**: Both should converge to the same eigenvalues, but may take different numbers of iterations
- **Thread-safety**: Arnoldi is thread-safe and re-entrant; ARPACK (Fortran) is not
- **Modern techniques**: Arnoldi incorporates more recent algorithmic improvements
